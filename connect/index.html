Contribute to Our Projects

We welcome contributions. See CONTRIBUTING.md. Quick start:

  1. Fork the repository — create a personal copy on your GitHub account.
  2. Create a branch — develop your feature or fix on a separate branch.
  3. Make your changes — follow the project’s conventions in CONTRIBUTING.md.
  4. Open a pull request — propose your changes for review.

Starter Issues

  • Write a Day-1 roadmap entry

    Pick a topic from the 7-day roadmap and write a short, opinionated explainer with 2–3 curated links.

  • Add a paper summary

    Summarise a classic paper (e.g. Almgren–Chriss, Black–Scholes) in plain English with intuition and a worked example.

  • Improve site accessibility

    Audit a page with Lighthouse and open a PR fixing the highest-impact issue.
